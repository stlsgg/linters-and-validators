Что такое ESLint?

ESLint — это статический анализатор кода для JavaScript, который помогает находить проблемные участки кода, не соответствующие заданным правилам.

В отличие от валидаторов, которые проверяют синтаксис, ESLint анализирует:

  • Стиль кодирования
  • Потенциальные ошибки
  • Проблемы с архитектурой
  • Неиспользуемый код
  • Сложность функций

Зачем нужен ESLint?

Предотвращение ошибок

Находит типичные ошибки до запуска кода: необъявленные переменные, неиспользуемые импорты, dead code.

Единый стиль кода

Гарантирует, что вся команда пишет код в одном стиле, независимо от предпочтений разработчиков.

Качество кода

Помогает избегать антипаттернов и писать более поддерживаемый код.

Экономия времени

Автоматически исправляет многие проблемы, избавляя от рутины.

Популярные правила ESLint

ESLint предлагает сотни правил. Вот самые полезные из них:

Правило Категория Описание Пример ошибки Исправление
no-unused-vars Возможные ошибки Запрещает неиспользуемые переменные const x = 10; (x не используется) Удалить переменную или использовать
eqeqeq Лучшие практики Требует строгого равенства (=== вместо ==) if (a == b) {} if (a === b) {}
semi Стиль Требует/запрещает точку с запятой const a = 1 (нет ;) const a = 1;
quotes Стиль Определяет тип кавычек const msg = "hello"; const msg = 'hello';
no-console Лучшие практики Запрещает console.log в продакшене console.log('debug'); Удалить или использовать логгер
indent Стиль Контролирует отступы function() {\nconsole.log();\n} function() {\n console.log();\n}
comma-dangle Стиль Висячие запятые в объектах/массивах const obj = {a: 1}; const obj = {a: 1,};
prefer-const ES6 Заменяет let на const если переменная не переопределяется let x = 5; const x = 5;

Быстрая настройка

Установка

npm install eslint --save-dev
# или
yarn add eslint --dev

Инициализация конфигурации

npx eslint --init

Мастер задаст вопросы о вашем проекте и создаст файл .eslintrc.js.

Пример конфигурации

module.exports = {
    env: {
        browser: true,
        es2021: true
    },
    extends: [
        'eslint:recommended'
    ],
    parserOptions: {
        ecmaVersion: 12,
        sourceType: 'module'
    },
    rules: {
        'semi': ['error', 'always'],
        'quotes': ['error', 'single']
    }
};

Интеграция с редакторами

VS Code

Установите расширение ESLint. Ошибки будут подсвечиваться прямо в редакторе.

WebStorm

Встроенная поддержка ESLint. Включите в Settings → Languages & Frameworks → JavaScript → Code Quality Tools → ESLint.

Автоисправление

Большинство редакторов поддерживают автоисправление при сохранении:

// VS Code - добавить в settings.json
"editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
}

Продвинутое использование

Популярные конфигурации

  • Airbnb — строгий и полный стиль
  • Standard — без точек с запятой, минимальная конфигурация
  • Google — стиль Google для JavaScript

Интеграция с Prettier

Используйте eslint-config-prettier для отключения правил ESLint, которые конфликтуют с Prettier.

Плагины для фреймворков

  • eslint-plugin-react — правила для React
  • eslint-plugin-vue — правила для Vue.js
  • @typescript-eslint/eslint-plugin — правила для TypeScript

ESLint — не просто инструмент, а стандарт индустрии. Начните использовать его сегодня, чтобы писать более чистый и надежный JavaScript код.
